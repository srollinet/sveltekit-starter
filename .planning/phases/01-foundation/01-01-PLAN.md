**Step 1 — Install adapter-node:**
```bash
cd /workspaces/sveltekit-starter
pnpm add -D @sveltejs/adapter-node@^5.5.4
```

**Step 2 — Update svelte.config.js:**
Replace whatever adapter the scaffold generated (likely `adapter-auto`) with `adapter-node`. The file must look like:

```js
import adapter from '@sveltejs/adapter-node';
import { vitePreprocess } from '@sveltejs/vite-plugin-svelte';

/** @type {import('@sveltejs/kit').Config} */
const config = {
  preprocess: vitePreprocess(),
  kit: {
    adapter: adapter(),
  },
};

export default config;
```

Do NOT import from `@sveltejs/adapter-auto` — that is a forbidden pattern per project constraints.

**Step 3 — Verify TypeScript strict mode in tsconfig.json:**
The scaffold generates a tsconfig.json that extends `.svelte-kit/tsconfig.json`. Check that `compilerOptions` contains `"strict": true`. The SvelteKit skeleton typically sets this already; if it doesn't, add it explicitly:

```json
{
  "extends": ".svelte-kit/tsconfig.json",
  "compilerOptions": {
    "allowJs": true,
    "checkJs": true,
    "esModuleInterop": true,
    "forceConsistentCasingInFileNames": true,
    "resolveJsonModule": true,
    "skipLibCheck": true,
    "sourceMap": true,
    "strict": true
  }
}
```

`"strict": true` enables: noImplicitAny, strictNullChecks, strictFunctionTypes, strictPropertyInitialization, noImplicitThis, alwaysStrict — this is required by FOUND-01.

**Step 4 — Run sync to generate .svelte-kit/tsconfig.json:**

```bash
cd /workspaces/sveltekit-starter
pnpm run check
```

This triggers `svelte-kit sync` which generates `.svelte-kit/tsconfig.json` that tsconfig.json extends. Verify it completes without errors.
</action>
