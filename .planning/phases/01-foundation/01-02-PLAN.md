  <action>
Install Tailwind CSS v4 and DaisyUI v5, and the Vite plugin for Tailwind v4.

**Step 1 — Install packages:**

```bash
cd /workspaces/sveltekit-starter
pnpm add -D tailwindcss@^4.2.2 @tailwindcss/vite@^4.2.2
pnpm add daisyui@^5.5.19
```

Note: Tailwind v4 uses `@tailwindcss/vite` as its Vite integration — not the PostCSS plugin used in v3. DaisyUI v5 is a runtime dependency (not dev-only) because it provides CSS loaded at runtime.

**Step 2 — Add Tailwind v4 Vite plugin to vite.config.ts:**
Read the existing vite.config.ts (scaffold-generated). Add `@tailwindcss/vite` as the FIRST plugin in the plugins array:

```typescript
import { sveltekit } from '@sveltejs/kit/vite';
import tailwindcss from '@tailwindcss/vite';
import { defineConfig } from 'vite';

export default defineConfig({
  plugins: [tailwindcss(), sveltekit()],
});
```

The Tailwind Vite plugin must come BEFORE sveltekit() in the plugins array.

Do NOT add tailwindcss as a PostCSS plugin in postcss.config.js — that is the v3 approach. Tailwind v4 uses the Vite plugin exclusively.
</action>

<task type="auto">
  <name>Task 2: Create CSS-first app.css and update app.html</name>
  <files>

  <action>
**Step 1 — Create src/app.css with CSS-first Tailwind v4 + DaisyUI v5 config:**

Write `src/app.css` with exactly this content (CSS-first approach, no tailwind.config.js):

```css
@import 'tailwindcss';
@plugin "daisyui" {
  themes:
    light --default,
    dark;
}
```

The `@plugin "daisyui"` directive with `themes: light --default, dark` configures DaisyUI v5 to include only the `light` and `dark` themes, with `light` as the default (per D-07). This is the CSS-first plugin configuration syntax in DaisyUI v5.

Do NOT use any of these forbidden patterns:

- `@tailwind base;`
- `@tailwind components;`
- `@tailwind utilities;`
- A `tailwind.config.js` file

**Step 2 — Update src/app.html:**
The scaffold's app.html needs two changes:

1. Add `data-theme="light"` to the `<html>` element (DaisyUI v5 reads this attribute to apply themes)
2. Ensure `%sveltekit.head%` is in `<head>` (scaffold already does this)

The final app.html should look like:

```html
<!doctype html>
<html lang="en" data-theme="light">
  <head>
    <meta charset="utf-8" />
    <link rel="icon" href="%sveltekit.assets%/favicon.png" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
    %sveltekit.head%
  </head>
  <body data-sveltekit-preload-data="hover">
    <div style="display: contents">%sveltekit.body%</div>
  </body>
</html>
```

The `data-theme="light"` on `<html>` sets the initial DaisyUI theme. The theme toggle (built in Plan 03) will change this attribute dynamically via JavaScript.

**Step 3 — Ensure app.css is imported in the root layout:**
Check if the scaffold-generated `src/routes/+layout.svelte` already imports `../app.css`. If not, the import will be added in Plan 03 when the layout is rewritten. Do not create a new layout here — just note this dependency.

**Step 4 — Verify Tailwind + DaisyUI resolve:**

```bash
cd /workspaces/sveltekit-starter
pnpm run build 2>&1 | tail -20
```

The build should succeed without CSS errors. If there are module resolution errors for `tailwindcss` or `daisyui`, re-check the Vite plugin installation from Task 1.
