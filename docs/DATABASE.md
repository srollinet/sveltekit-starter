# Database

This project uses [Drizzle ORM](https://orm.drizzle.team) with PostgreSQL.

## Schema

- Schema files live in `src/lib/server/db/schema/` — **one file per table**.
- Re-export all tables from `src/lib/server/db/schema/index.ts`.
- Use `uuidv7()` for primary keys (application-generated, not DB-generated).
- Always add `withTimezone: true` to timestamp columns.
- Use `pgEnum` for columns with a fixed set of values.

```ts
import { pgEnum, pgTable, text, timestamp, uuid } from 'drizzle-orm/pg-core';
import { uuidv7 } from 'uuidv7';

export const statusEnum = pgEnum('status', ['draft', 'published']);

export const posts = pgTable('posts', {
  id: uuid('id')
    .primaryKey()
    .$defaultFn(() => uuidv7()),
  title: text('title').notNull(),
  status: statusEnum('status').notNull().default('draft'),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true })
    .notNull()
    .defaultNow()
    .$onUpdateFn(() => new Date()),
});
```

## Queries — always project, never return raw rows

Never return full table rows to the client. Select only the columns needed.

```ts
// Select specific columns (returns a plain object, safe to send to client)
const results = await db
  .select({ id: posts.id, title: posts.title, status: posts.status })
  .from(posts)
  .where(eq(posts.status, 'published'));
```

## Workflow

1. Edit schema files in `src/lib/server/db/schema/`
2. Run `pnpm db:generate` — creates a new migration file in `drizzle/`
3. Run `pnpm db:migrate` — applies pending migrations to the database
4. Commit both schema changes and the generated migration file

> `pnpm db:push` pushes schema directly without a migration file — **dev only, never in production**.

## Migrations

```bash
pnpm db:generate   # generate a new migration file from schema changes
pnpm db:migrate    # apply pending migrations to the database
pnpm db:push       # push schema directly (no migration file — dev only)
pnpm db:studio     # open Drizzle Studio to browse data
```

Always commit migration files generated by `db:generate`.
